# Яндекс.Вебмастер: индексация

В проекте уже сделано:

- В корне есть **`.env`** с `VITE_SITE_URL=https://ses88.ru` (не коммитьте в git — файл в `.gitignore`).
- При **`vite build`** без `.env` всё равно генерируются **`robots.txt`** и **`sitemap.xml`** для **https://ses88.ru**.
- В SPA при смене страницы обновляются **canonical** и **og:url**, отправляется **хит Метрики** (счётчик в `index.html`).

## 1. Подтверждение прав на сайт

1. Зайдите в [Яндекс.Вебмастер](https://webmaster.yandex.ru/) → добавьте сайт.
2. Выберите способ **«HTML-тег»** (meta).
3. Скопируйте **код подтверждения** (значение `content` у meta `yandex-verification`).
4. В корне проекта создайте файл `.env` (можно скопировать из `.env.example`) и укажите:
   ```env
   VITE_YANDEX_VERIFICATION=ваш_код_из_вебмастера
   ```
5. Пересоберите фронт (`npm run build` в каталоге с Vite) и задеплойте. В `<head>` попадёт мета-тег подтверждения.
6. В Вебмастере нажмите **«Проверить»**.

Без кода в `.env` мета-тег не добавляется — это нормально для локальной разработки.

## 2. Домен для sitemap и robots.txt

При **production-сборке** создаются файлы `robots.txt` и `sitemap.xml` в папке сборки (`dist/public`), если задан адрес сайта:

```env
VITE_SITE_URL=https://ses88.ru
```

Без `VITE_SITE_URL` файлы **не генерируются** (в консоли будет предупреждение). В коде клиента по умолчанию используется тот же домен (`client/src/siteConfig.ts`).

После деплоя проверьте в браузере:

- `https://ses88.ru/robots.txt`
- `https://ses88.ru/sitemap.xml`

## 3. Указать sitemap в Вебмастере

1. **Индексирование** → **Файлы Sitemap**.
2. Добавьте URL: `https://ses88.ru/sitemap.xml`.

## 4. Счётчик Метрики

Счётчик Яндекс.Метрики подключён в `client/index.html`. С Вебмастером он не связан напрямую, но полезен для аналитики.

## 5. Обновление списка страниц в sitemap

Список URL задаётся в `vite.config.ts` в массиве `SITEMAP_PATHS`. При добавлении новых публичных маршрутов допишите пути туда и пересоберите проект.
